# Security Policy

## Supported Versions

The following versions of this repository are currently supported with security updates.

| Version              | Supported          |
| -------------------- | ------------------ |
| Latest (main branch) | :white_check_mark: |
| Older versions       | :x:                |

Security fixes and updates will only be provided for the latest version of this project.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly.

**Contact Email:**
[rajeevsharmamachphy@gmail.com](mailto:rajeevsharmamachphy@gmail.com)

### How to Report

Please include the following information in your report:

* Description of the vulnerability
* Steps to reproduce the issue
* Proof of Concept (PoC) if available
* Affected file or module
* Potential impact of the vulnerability

### Response Timeline

* Initial response: within **48–72 hours**
* Investigation and verification: **up to 7 days**
* Fix and patch release (if confirmed): **as soon as possible**

### Responsible Disclosure

Please **do not publicly disclose** the vulnerability until it has been reviewed and fixed. Responsible disclosure helps protect users and maintainers of this repository.

## Scope

This repository is intended for **security research and educational purposes only**.
Any tools, scripts, or techniques provided here should only be used in:

* Authorized security testing environments
* Personal labs
* Capture The Flag (CTF) platforms
* Bug bounty programs where testing is permitted

Unauthorized use of these techniques against systems without permission may be illegal.
